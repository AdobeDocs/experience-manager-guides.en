# Working with PDF engine v2 in Experience Manager Guides

PDF Engine V2 introduces an upgraded PDF generation library with improved rendering capabilities, support for modern OpenType fonts, and enhanced CSS processing. As a result, PDFs generated using PDF Engine V2 may differ slightly from PDFs generated using PDF Engine V1.

[Enable PDF engine v2]().

PDF Engine V2 is based on an upgraded PDF generation library and includes changes to font handling, CSS processing, and PDF rendering behavior. As a result, the appearance of PDFs generated using PDF Engine V2 may differ from PDFs generated using PDF Engine V1.

For example, PDF Engine V2 supports newer OpenType fonts, whereas PDF Engine V1 relied on TrueType fonts. Similar enhancements in the rendering engine can affect text layout, spacing, styling, and overall document appearance.

To help maintain output consistency after migration, you may need to update your custom CSS. Depending on the styling features used in your PDF templates, specific CSS changes can help align the PDF output generated by PDF Engine V2 with the output previously produced by PDF Engine V1.

For information about the recommended CSS updates, see CSS changes to maintain PDF Engine V1 output in PDF Engine V2.

## CSS changes to maintain PDF Engine V1 output

The following table describes rendering and behavior changes introduced in PDF Engine V2 and the corresponding CSS updates that can help maintain output consistency.

| Description | Changes needed to maintain PDF Engine V1 output |
|-------------|------------------------------------------------|
|The image rendering algorithm has changed in PDF Engine V2. This may result in visual differences when scaling or rendering images in generated PDFs compared to PDF Engine V1. | To restore the image rendering behavior used by PDF Engine V1, add the following CSS rule to your custom stylesheet: <br> ```css * { image-rendering: pixelated; }``` |
|Text spacing and line wrapping may differ in PDF Engine V2 due to updates in font rendering and glyph layout processing. These changes can affect character spacing, word spacing, and text flow compared to PDFs generated using PDF Engine V1| If your stylesheet uses the sans-serif font family or fonts that exhibit spacing differences after migration, add the following CSS property to use the glyph layout behavior that more closely matches PDF Engine V1 output: <br> ```css body { -ro-glyph-layout-mode: quality; }```|
|Footnote references may no longer appear as superscript markers in PDF Engine V2. Changes to the default footnote marker styling can affect the appearance and formatting of footnote references in generated PDFs.|To restore the superscript-style footnote markers used in PDF Engine V1, define a custom style for the `::footnote-marker`pseudo-element:<br> ```css .fn::footnote-marker { content: counter(footnote) " "; vertical-align: super; font-size: 65%; }```|
